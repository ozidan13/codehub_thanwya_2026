LECTURE 02

HTML من الألف للياء

HyperText Markup Language — الدليل الشامل

محاضرة تفصيلية تغطي كل حاجة محتاج تعرفها عن HTML — من أول إنشاء أول صفحة ويب لحد Semantic HTML و Accessibility و SEO.

ليه HTML مهمة أصلاً؟

HTML هي اللغة الأساسية اللي كل صفحة ويب في العالم مبنية بيها. مفيش موقع واحد على الإنترنت — لا فيسبوك، ولا جوجل، ولا يوتيوب — إلا وبيستخدم HTML.

لما بتفتح أي صفحة ويب، الـ browser بتاعك (زي Chrome أو Firefox) بياخد ملف HTML من الـ server وبيحوله لـ visual page قدامك. الملف ده بيقول للـ browser: "هنا عنوان، وهنا فقرة، وده لينك، ودي صورة." يعني HTML هي اللغة اللي بتوصف هيكل ومحتوى الصفحة.

فكر فيها كده: لو الموقع عمارة، HTML هي الخرسانة والطوب — الهيكل اللي كل حاجة بتتبني عليه. CSS هي الدهان والديكور، و JavaScript هي الكهربا والأجهزة اللي بتخلي كل حاجة تتحرك وتشتغل. من غير HTML مفيش عمارة أصلاً.

إزاي الـ Browser بيشتغل؟

1. User types URL

2. Server sends HTML

3. Browser parses HTML

4. Page rendered

لما بتكتب عنوان موقع في الـ browser، الموضوع بيمشي كده: الـ browser بيروح للـ server ويقوله "عايز الصفحة دي"، الـ server بيبعتله ملف HTML، الـ browser بيقراه سطر سطر وبيحوله للشكل اللي بتشوفه. العملية دي بتحصل في ملي ثانية!

الهيكل الأساسي لملف HTML

The Essential HTML Document Structure

أول ملف HTML

أي ملف HTML لازم يبدأ بـ <!DOCTYPE html> — ده مش tag عادي، ده تعليمات للـ browser إن الملف ده مكتوب بـ HTML5 (آخر إصدار). بعد كده بييجي هيكل ثابت كل ملف HTML بيتبعه:

index.html 
<!-- ده أول سطر في أي ملف HTML -->
<!DOCTYPE html>
<html lang="ar" dir="rtl">

  <head>
    <!-- معلومات عن الصفحة (مش بتظهر للمستخدم) -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>أول صفحة ويب</title>
  </head>

  <body>
    <!-- المحتوى اللي هيظهر في الصفحة -->
    <h1>أهلاً بيك في أول صفحة!</h1>
    <p>دي أول فقرة في صفحتي.</p>
  </body>

</html>

النتيجة في الـ Browser
<!DOCTYPE html>

بيقول للـ browser: "أنا ملف HTML5". لازم يكون أول سطر.

<html lang="ar">

الـ root element — كل حاجة تانية جواه. الـ lang بيحدد لغة الصفحة.

<head>

فيه metadata — معلومات مش بتظهر في الصفحة (العنوان، الوصف، CSS links).

<body>

كل حاجة المستخدم بيشوفها — العناوين، الفقرات، الصور، اللينكات — كلها جوا الـ body.

كده ببساطة: الـ <head> زي بطاقة الهوية بتاعتك — معلومات عنك بس الناس مش بتشوفها أول ما تقابلك. الـ <body> هو أنت نفسك — اللي الناس بتشوفه وبتتعامل معاه.

الـ Meta Tags المهمة

الـ meta tags دي معلومات إضافية بتحطها جوا الـ <head>. مهمة جداً للـ SEO (ظهور موقعك في جوجل) وللـ responsive design (إن الموقع يظهر كويس على الموبايل).

meta-tags.html 
<head>
  <!-- ❶ الترميز: عشان الحروف العربية تظهر صح -->
  <meta charset="UTF-8">

  <!-- ❷ الـ Viewport: عشان الموقع يظهر كويس على الموبايل -->
  <meta name="viewport"
        content="width=device-width, initial-scale=1.0">

  <!-- ❸ وصف الصفحة: بيظهر في نتائج جوجل -->
  <meta name="description"
        content="تعلم HTML من الصفر — دروس بالعربي">

  <!-- ❹ الأيقونة اللي بتظهر في الـ Tab -->
  <link rel="icon" href="favicon.ico">

  <!-- ❺ Open Graph: لما حد يشير الموقع على فيسبوك -->
  <meta property="og:title" content="تعلم HTML">
  <meta property="og:image" content="preview.jpg">

  <title>تعلم HTML</title>
</head>
نصيحة مهمة: لو أنت بتعمل موقع بالعربي، لازم تحط charset="UTF-8" عشان الحروف العربية تظهر صح ومتبوظش. ولو نسيت الـ viewport، موقعك هيبان صغير أوي على الموبايل.
02

العناصر والتاجات

HTML Elements, Tags & Nesting Rules

يعني إيه Element؟

الـ Element هو الوحدة الأساسية في HTML. كل element بيتكون من 3 حاجات: Opening Tag + Content + Closing Tag.

<p class="intro">Hello World</p>
↑ Opening Tag ↑ Attribute ↑ Content ↑ Closing Tag
فكر في الـ element زي ساندويتش: العيشة الفوقانية هي الـ opening tag، الحشو هو الـ content، والعيشة التحتانية هي الـ closing tag. بعض الـ tags زي <img> مبيحتاجوش closing tag — دول اسمهم self-closing أو void elements.

Block vs Inline Elements

Block Elements

بياخد الـ width كله وبيبدأ في سطر جديد. زي الصناديق الكبيرة اللي كل واحد فوق التاني.

block-elements.html
<!-- كل دي block elements -->
<div>...</div>
<p>...</p>
<h1>...</h1>
<section>...</section>
<ul>...</ul>

Inline Elements

بياخد مساحة المحتوى بتاعه بس ومش بيعمل سطر جديد. بيكون جوا الـ block element.

inline-elements.html
<!-- كل دي inline elements -->
<span>...</span>
<a href="#">...</a>
<strong>...</strong>
<em>...</em>
<img src="...">
خطأ شائع: متحطش block element جوا inline element! يعني <span><div>...</div></span> ده غلط. الـ inline بيكون جوا الـ block مش العكس.

الـ Void Elements (Self-closing)

فيه elements معندهاش content ومش بيحتاجوا closing tag. دول اسمهم void elements. أشهرهم:

Element الوظيفة مثال
<img> صورة <img src="photo.jpg" alt="وصف">
<br> سطر جديد سطر<br>سطر تاني
<hr> خط فاصل <hr>
<input> حقل إدخال <input type="text">
<meta> بيانات وصفية <meta charset="UTF-8">
<link> ربط ملفات <link rel="stylesheet" href="style.css">
دول عناصر "مكتفية بذاتها" — مش محتاجة تقفلها. الـ <img> مثلاً مش منطقي يكون جواها content، هي نفسها المحتوى.
03

الـ Attributes بالتفصيل

HTML Attributes — Global, Boolean & Data Attributes

يعني إيه Attribute؟

الـ Attributes هي معلومات إضافية بتحطها جوا الـ opening tag عشان تتحكم في سلوك أو شكل الـ element. كل attribute بيتكون من name="value".

attributes.html 
<!-- href هو الـ attribute اللي بيحدد لينك الصفحة -->
<a href="https://google.com" target="_blank">اضغط هنا</a>

<!-- src بيحدد مصدر الصورة، و alt بيوصفها -->
<img src="photo.jpg" alt="صورة منظر طبيعي" width="600">

<!-- id و class: أهم attributes للتعامل مع CSS و JavaScript -->
<div id="main-content" class="container active">
  محتوى الصفحة
</div>

الـ Global Attributes

دول attributes ينفع تستخدمهم مع أي HTML element:

id="unique-name"

اسم فريد للـ element — مش ينفع يتكرر في نفس الصفحة. بتستخدمه مع CSS و JavaScript.

class="name1 name2"

اسم تصنيفي — ينفع يتكرر ويتحط أكتر من class. الأساس في CSS styling.

style="color: red;"

Inline CSS — بتحط styling مباشر على الـ element. مش مستحب في الشغل الحقيقي.

title="tooltip text"

نص بيظهر لما تحط الماوس فوق الـ element (tooltip).

data-*="custom"

Custom attributes بتعملها بنفسك لخزن data إضافية. بتقراها من JavaScript.

hidden

Boolean attribute — بيخفي الـ element من الصفحة. مش محتاج قيمة.

القاعدة الذهبية: الـ id زي رقم البطاقة — فريد لكل شخص. الـ class زي اسم المدرسة — ناس كتير ممكن يكونوا نفس المدرسة.

الـ Data Attributes في الشغل الحقيقي

الـ data-* attributes مفيدة جداً لما تيجي تربط HTML بـ JavaScript. مثال حقيقي من e-commerce:

product-card.html 
<!-- كارت منتج في موقع تسوق -->
<div class="product-card"
     data-product-id="12345"
     data-price="299.99"
     data-category="electronics">

  <h3>Samsung Galaxy S24</h3>
  <button class="add-to-cart">أضف للسلة</button>
</div>

<!-- في JavaScript بتقرا الـ data كده: -->
<script>
  const card = document.querySelector('.product-card');
  console.log(card.dataset.productId);  // "12345"
  console.log(card.dataset.price);      // "299.99"
</script>
ده pattern شائع جداً في الشغل الحقيقي. بدل ما تعمل API call كل مرة، بتخزن الـ data في الـ HTML نفسه وتقراها بـ JavaScript لما تحتاجها.

العناوين (h1 — h6)

تنسيق النصوص

HTML فيها tags كتير لتنسيق النصوص. كل tag ليه معنى دلالي (semantic meaning) مش مجرد شكل:

Tag الشكل المعنى الدلالي مثال عملي
<strong> Bold أهمية قوية (مش مجرد bold) تحذير: ممنوع الدخول
<em> Italic تأكيد / stress ده مش اختياري
<mark> Highlight نص مميز / highlight ابحث عن كلمة HTML
<del> Deleted نص تم حذفه السعر: 500 300 جنيه
<code> Code كود برمجي استخدم console.log()
<abbr> HTML اختصار مع tooltip API
<blockquote> Quote اقتباس طويل اقتباس من مصدر تاني
فيه فرق بين <strong> و <b>! الاتنين شكلهم bold بس <strong> بيقول للـ screen reader "ده كلام مهم اقراه بتركيز"، لكن <b> مجرد شكل. دايماً استخدم الـ semantic tag.

الروابط والصور والميديا

Links, Images, Audio & Video

الروابط — الـ Anchor Tag

الـ <a> tag هو اللي بيعمل hyperlinks — وده أساس الويب كله. "HyperText" في HTML معناها "نص مرتبط بنصوص تانية". الـ attribute الأساسي هو href (Hypertext REFerence).

links.html 
<!-- ❶ لينك لموقع خارجي -->
<a href="https://google.com" target="_blank" rel="noopener">
  فتح جوجل في تاب جديد
</a>

<!-- ❷ لينك لصفحة تانية في نفس الموقع -->
<a href="about.html">من نحن</a>

<!-- ❸ لينك لقسم في نفس الصفحة (anchor) -->
<a href="#contact">روح لقسم الاتصال</a>

<!-- ❹ لينك لإرسال إيميل -->
<a href="mailto:info@example.com">ابعتلنا إيميل</a>

<!-- ❺ لينك لتحميل ملف -->
<a href="notes.pdf" download>حمل المذكرة</a>
أمان: لما تفتح لينك خارجي بـ target="_blank"، دايماً حط rel="noopener" عشان تمنع الصفحة الجديدة إنها تتحكم في صفحتك (ثغرة أمنية معروفة).

الصور — أكتر من مجرد <img>

<img src="photo.jpg"
     alt="وصف دقيق للصورة"
     width="800"
     height="600"
     loading="lazy">

<!-- ❷ صورة مع caption -->
<figure>
  <img src="chart.png" alt="Sales chart for Q4 2025">
  <figcaption>رسم بياني لمبيعات الربع الرابع 2025</figcaption>
</figure>

<!-- ❸ Responsive Image: صور مختلفة لأحجام شاشة مختلفة -->
<picture>

  <source media="(max-width: 1200px)" srcset="photo-medium.jpg">
  <img src="photo-large.jpg" alt="صورة responsive">
</picture>
الـ alt مش اختياري! لو مكتبتش alt، الناس اللي بتستخدم screen readers مش هتعرف إيه في الصورة. وكمان جوجل بيقرا الـ alt عشان يفهم الصورة. اكتب وصف دقيق ومفيد مش مجرد "صورة".

النماذج — Forms

Building Interactive Forms with HTML5 Validation

ليه الـ Forms مهمة؟

الـ Forms هي الطريقة الأساسية اللي المستخدم بيبعت بيها بيانات للـ server. كل مرة بتسجل في موقع، بتبحث في جوجل، أو بتكتب تعليق — أنت بتستخدم form. الـ <form> tag بياخد attributeين أساسيين: action (الصفحة اللي هتستقبل البيانات) و method (طريقة الإرسال: GET أو POST).

registration-form.html 
<form action="/api/register" method="POST">

  <!-- ❶ حقل نصي عادي -->
  <label for="username">اسم المستخدم:</label>
  <input type="text" id="username" name="username"
         required minlength="3" maxlength="20"
         placeholder="مثال: ahmed_99">

  <!-- ❷ إيميل — الـ browser هيتحقق تلقائياً -->
  <label for="email">البريد الإلكتروني:</label>
  <input type="email" id="email" name="email" required>

  <!-- ❸ باسورد -->
  <label for="pass">كلمة السر:</label>
  <input type="password" id="pass" name="password"
         required minlength="8"
         pattern="(?=.*\d)(?=.*[a-z]).{8,}">

  <!-- ❹ قائمة منسدلة -->
  <label for="country">البلد:</label>
  <select id="country" name="country">
    <option value="">اختر بلدك</option>
    <option value="eg">مصر</option>
    <option value="sa">السعودية</option>
  </select>

  <!-- ❺ مربع نص كبير -->
  <label for="bio">نبذة عنك:</label>
  <textarea id="bio" name="bio" rows="4"></textarea>

  <!-- ❻ Checkbox -->
  <label>
    <input type="checkbox" name="terms" required>
    أوافق على الشروط والأحكام
  </label>

  <button type="submit">تسجيل</button>
</form>
لاحظ: الـ required و pattern و minlength دول HTML5 validation — الـ browser بيتحقق منهم تلقائياً من غير JavaScript! بس متعتمدش عليهم و بس — لازم يكون فيه validation في الـ server كمان (defense in depth).

Semantic HTML بالتفصيل

Meaningful Markup — Beyond Divs and Spans

يعني إيه Semantic؟

Semantic HTML معناها إنك تستخدم tags بتوصف معنى المحتوى مش مجرد شكله. بدل ما تحط كل حاجة في <div>، تستخدم tags معبرة زي <nav> للقائمة و <article> للمقال.

<header>
<nav>
<main>
<article>
<section>
<aside>
<footer>
semantic-layout.html 
<body>
  <header>
    <h1>اسم الموقع</h1>
    <nav>
      <ul>
        <li><a href="/">الرئيسية</a></li>
        <li><a href="/about">من نحن</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h2>عنوان المقال</h2>
      <p>محتوى المقال...</p>
      <time datetime="2026-02-16">16 فبراير 2026</time>
    </article>

    <section>
      <h2>التعليقات</h2>
      <!-- التعليقات هنا -->
    </section>
  </main>

  <aside>
    <h3>مقالات ذات صلة</h3>
  </aside>

  <footer>
    <p>&copy; 2026 جميع الحقوق محفوظة</p>
    <address>تواصل معنا: info@site.com</address>
  </footer>
</body>
ليه دي مهمة؟ ① جوجل بيفهم صفحتك أحسن = SEO أحسن. ② Screen readers بتقدر تنقل المستخدمين للـ nav أو الـ main مباشرة. ③ الكود بيكون أسهل في القراية والصيانة. الصفحة اللي كلها <div> زي كتاب من غير فهرس!

09

Accessibility — الوصولية

Making the Web Usable for Everyone (a11y)

يعني إيه a11y؟

Accessibility (بنختصرها a11y — لأن فيه 11 حرف بين الـ a والـ y) معناها إن موقعك يكون قابل للاستخدام من كل الناس — حتى اللي عندهم إعاقة بصرية أو حركية. مش بس ده الصح أخلاقياً، ده كمان في كتير من الدول مطلوب قانوناً.

أعمل ده ✅

  • ✅ حط alt لكل صورة مش decorative
  • ✅ استخدم <label> مع كل input
  • ✅ استخدم semantic tags مش كل حاجة div
  • ✅ خلي الموقع يشتغل بالـ keyboard
  • ✅ حط lang attribute على الـ html
  • ✅ اعمل skip link للمحتوى الرئيسي

متعملش كده ❌

  • ❌ صور من غير alt
  • ❌ لينكات بتقول "اضغط هنا" بس
  • ❌ ألوان contrast ضعيف
  • ❌ إزالة الـ focus outline
  • ❌ عناوين الصفحة متكررة أو فاضية
  • ❌ فيديو من غير subtitles

ARIA Attributes

لما الـ semantic tags مش كافية، بنستخدم ARIA (Accessible Rich Internet Applications) عشان نوصف العناصر للـ screen readers:

aria-examples.html 
<!-- ❶ Skip Link: أول حاجة في الـ body -->
<a href="#main-content" class="skip-link">
  اذهب للمحتوى الرئيسي
</a>

<!-- ❷ زرار بأيقونة بس (من غير نص) -->
<button aria-label="إغلاق القائمة">
  <i class="fas fa-times"></i>
</button>

<!-- ❸ حالة عنصر متغيرة -->
<button aria-expanded="false">عرض التفاصيل</button>
<div aria-hidden="true">محتوى مخفي</div>

<!-- ❹ صورة decorative (مش محتاجة وصف) -->
<img src="decoration.svg" alt="" role="presentation">
القاعدة الأولى في ARIA: لو تقدر تستخدم semantic HTML tag، متستخدمش ARIA! مثلاً: استخدم <button> بدل <div role="button">. الـ semantic tag بيدي accessibility مجاناً.

أخطاء HTML شائعة

❌ نسيان إقفال الـ tags

<p>فقرة أولى <p>فقرة تانية ← غلط! وين الـ </p>؟

❌ تداخل خاطئ (Nesting)

<strong><em>نص</strong></em> ← غلط! لازم تقفل الداخلي الأول

❌ استخدام <br> للـ spacing

الـ <br> للسطر الجديد مش للمسافات. استخدم CSS margin/padding.

❌ استخدام inline styles بكثرة

بدل style="color:red" على كل element، اعمل CSS class واحد واستخدمه.
